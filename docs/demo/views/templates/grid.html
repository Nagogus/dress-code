To use several cards in one line, use .dc-box--bleed to offset the box padding and align the columns to the general grid.

Responsive behavior

By default, the grid is fluid: the columns and gutters resize to fit the screen size but each element's column width stay the same. It's possible to make layout responsive by adding utility classes to make elements have different column-widths or to hide them at specific viewport widths.

There are 7 viewport widths, named tiny, small, medium, large, huge, giant, monstrous.

Adding screen-width-specific exceptions

Using the dc-[viewport-width]-span[x] modifiers you can add exceptions to the default element size, that activate only at specific screen sizes. In the example below, the default sizes are overriden for tiny, small, giant and monstrous.

Note: when creating size exceptions, remember to add size specific -last classes, e.g. dc-tiny-last.
